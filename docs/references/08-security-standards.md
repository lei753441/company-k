# セキュリティ基準・ガイドライン

---

## NIST SP 800-63B（デジタルアイデンティティガイドライン）

### 概要

| 項目 | 内容 |
|------|------|
| 発行元 | 米国国立標準技術研究所（NIST） |
| 正式名称 | Digital Identity Guidelines: Authentication and Lifecycle Management |
| 版 | SP 800-63B（2017年、2024年改訂） |
| 参照根拠 | 国際的なセキュリティ標準として国内でも広く参照 |

---

### 本システムへの主な適用要件（01-auth-access.md）

#### パスワードポリシー

| 要件 | NIST推奨 | 本システムへの適用 |
|------|---------|-----------------|
| 最小文字数 | 8文字以上 | 12文字以上を推奨 |
| 定期変更強制 | **不要**（漏洩時のみ強制リセット） | 定期変更強制なし |
| 複雑性要件 | 過度な複雑性要件は設けない | 英字+数字の組み合わせのみ要求 |
| 漏洩パスワード確認 | 既知の漏洩パスワードリストとの照合 | Have I Been Pwned API等を検討 |
| ヒントの禁止 | パスワードヒントは設けない | 実装しない |

> **重要**: 「90日ごとのパスワード変更強制」はNIST SP 800-63Bで**非推奨**とされている。ユーザーが単純なパスワードを使い回すリスクが増大するため。

#### 多要素認証（MFA）

| ロール | MFA要件 |
|--------|--------|
| 管理者 | **必須** |
| 事務 | **必須** |
| 営業 | 推奨（任意） |
| 社員 | 任意 |
| 協力会社/個人事業主 | 任意 |

#### セッション管理
- セッションタイムアウト：30分（操作なし）
- 最大セッション時間：8時間
- ログアウト時のセッション完全無効化

## 情報セキュリティ関連ガイドライン

### IPA「中小企業の情報セキュリティ対策ガイドライン」

IT会社として最低限実施すべき対策の参考：

| 対策 | 本システムへの適用 |
|------|-----------------|
| アクセス権の管理 | ロールベースアクセス制御（RBAC）実装 |
| パスワード管理 | 上記NIST基準に準拠 |
| ソフトウェア更新 | 依存ライブラリの定期的な脆弱性チェック |
| バックアップ | データの定期バックアップ（RTO・RPO定義） |
| インシデント対応 | 漏洩時の報告フロー・影響範囲調査手順 |

## 暗号化要件

### データ暗号化

| 対象データ | 暗号化方式 | 適用箇所 |
|-----------|----------|---------|
| パスワード | bcrypt（コストファクター12以上） | 認証管理 |
| マイナンバー | AES-256-GCM | 社員管理 |
| 銀行口座情報 | AES-256-GCM | 社員・協力会社管理 |
| 給与情報 | AES-256-GCM | 社員管理 |
| 通信経路 | TLS 1.2以上（TLS 1.3推奨） | 全通信 |

## 監査ログ要件

### 記録必須イベント

| イベント種別 | 記録内容 | 保存期間 |
|------------|---------|---------|
| ログイン成功/失敗 | ユーザーID・IPアドレス・タイムスタンプ | 7年 |
| マイナンバーアクセス | アクセス者・対象者・目的・日時 | 7年（法的要件） |
| 個人情報含有書類閲覧 | アクセス者・対象書類・日時 | 7年 |
| 権限変更 | 変更者・変更内容・対象ユーザー | 7年 |
| データ削除操作 | 操作者・削除対象・削除理由 | 7年 |
| 請求書・支払操作 | 操作者・対象・金額・日時 | 7年 |

> 監査ログ自体は**改ざん不可**（参照専用、削除権限なし）
