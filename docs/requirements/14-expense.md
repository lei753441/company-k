# 経費管理 - 要件定義

## 1. モジュール概要

### 1.1 目的
社員・協力会社が発生させた業務上の経費申請・承認・精算を一元的に管理する。領収書の電子化・保存、承認ワークフロー、月次精算処理を通じて経費業務の効率化と内部統制の強化を実現する。

### 1.2 対象ユーザー
| ロール | 主な操作 |
|---|---|
| 社員 | 経費申請、領収書アップロード、申請状況確認 |
| 協力会社/個人事業主 | 経費申請（案件費用）、領収書アップロード |
| 事務 | 経費承認、月次精算処理、集計 |
| 管理者 | 全データ参照・修正、カテゴリ・上限額設定 |
| 営業 | 担当案件の経費状況参照（読み取り専用） |

### 1.3 スコープ
- 社員・協力会社による経費申請・領収書電子保存
- 承認ワークフロー（申請→承認→精算）
- 経費カテゴリ別管理・上限額管理
- 月次精算処理
- 案件・プロジェクトへの費用紐付け
- 経費集計・レポート

---

## 2. 経費申請フロー（申請→承認→精算）

```
社員/協力会社が経費申請を入力
  ・経費カテゴリ選択
  ・金額・日付・目的入力
  ・領収書画像/PDFをアップロード
  ↓
自動チェック
  ・上限額チェック（カテゴリ別・月次別）
  ・必須項目チェック
  ・領収書添付チェック
  ↓
事務/管理者が承認
  （または差し戻し）
  ↓
承認済み経費の月次精算
  ↓
精算（銀行振込または給与計上）
  ↓
精算完了通知
```

### 2.1 ステータス定義
| ステータス | 説明 |
|---|---|
| 下書き | 入力中・未提出 |
| 申請中 | 提出完了・承認待ち |
| 差し戻し | 承認者が修正指示 |
| 承認済 | 承認完了・精算待ち |
| 精算済 | 月次精算処理完了 |
| 却下 | 申請却下 |

## 3. 機能要件

### 3.1 経費申請機能

#### 3.1.1 申請入力
- **経費カテゴリ選択**: 交通費・接待費・通信費等のカテゴリ選択（Section 4参照）
- **基本情報入力**: 発生日・金額（税抜/税込）・利用目的・摘要
- **案件紐付け**: 案件（プロジェクト）への紐付け（任意・案件費用として計上する場合は必須）
- **消費税率**: 10%または8%（軽減税率対象品目の場合）を選択
- **領収書添付**: 領収書の写真またはPDF（JPEG/PNG/PDF対応）をアップロード（一申請に複数枚対応）
- **下書き保存**: 入力途中での下書き保存
- **一括申請**: 交通費等、複数日分をまとめて1申請として登録可能

#### 3.1.2 自動チェック
- **上限額チェック**: カテゴリ別月次上限額を超過する場合は警告（申請は可能だが要注意表示）
- **領収書添付チェック**: 一定金額以上の申請は領収書必須（閾値は設定可能）
- **重複チェック**: 同一日付・同一金額・同一カテゴリの申請がある場合は警告
- **申請期限チェック**: 発生日から一定期間（デフォルト：2ヶ月）を経過した場合は警告

#### 3.1.3 申請一覧・状況確認
- **自身の申請一覧**: 申請日・カテゴリ・金額・ステータスで表示
- **申請履歴**: 過去の申請履歴を月単位で確認
- **承認・差し戻し通知**: ステータス変更時に通知（Team E連携）

### 3.2 領収書管理

#### 3.2.1 領収書アップロード
- **対応フォーマット**: JPEG/PNG/PDF
- **ファイルサイズ上限**: 1ファイルあたり10MB
- **複数枚対応**: 1申請に対して複数の領収書を添付可能
- **画像プレビュー**: アップロード後にサムネイル表示

#### 3.2.2 電子帳簿保存法対応
- **タイムスタンプ**: アップロード日時をシステムで記録（真実性の確保）
- **解像度確認**: 200dpi以上であることの確認（ガイダンス表示）
- **訂正・削除履歴**: 変更・削除操作の履歴を記録
- **検索機能**: 日付・金額・取引先名での検索（電子帳簿保存法 検索要件対応）
- **保存期間**: 7年間の保存（法人税法の要件）

### 3.3 承認機能

#### 3.3.1 承認一覧
- **申請一覧表示**: 申請者・申請日・カテゴリ・金額・ステータスで絞り込み
- **領収書確認**: アップロードされた領収書のプレビュー表示
- **コメント入力**: 承認・差し戻し時にコメントを入力可能

#### 3.3.2 承認操作
- **承認**: 申請内容を確認し承認。コメント入力可
- **差し戻し**: 修正理由を入力して差し戻し。申請者へ通知（Team E連携）
- **却下**: 不正・対象外経費として却下。理由入力必須
- **一括承認**: 複数件を選択して一括承認

#### 3.3.3 承認権限
- **通常承認**: 事務担当者が承認（一定金額未満）
- **管理者承認**: 設定金額以上は管理者承認が必要（二段階承認オプション）
- **閾値設定**: カテゴリ別・金額別の承認権限閾値を管理者が設定

### 3.4 月次精算機能

#### 3.4.1 精算処理
- **精算対象抽出**: 対象月の承認済み経費を一覧表示
- **精算方法**: 給与振込への上乗せ、または別途振込（社員別に設定）
- **精算明細**: 社員・協力会社別の精算額を集計
- **精算データ出力**: 給与システム連携用CSVまたは振込データ（全銀フォーマット）出力
- **精算完了通知**: 精算処理完了後に対象者へ通知（Team E連携）

#### 3.4.2 月次集計・レポート
- **社員別集計**: 社員・協力会社ごとの月次経費合計
- **カテゴリ別集計**: 経費カテゴリ別の月次集計
- **案件別集計**: 案件・プロジェクトへの費用配賦集計（Team C連携）
- **CSVエクスポート**: 集計データのCSV出力
- **Team Eダッシュボード連携**: 月次経費サマリーデータを提供

### 3.5 管理者向け設定機能
- **カテゴリ管理**: 経費カテゴリの追加・編集・無効化
- **上限額設定**: カテゴリ別・月次上限額の設定
- **承認閾値設定**: 承認権限の金額閾値設定
- **領収書必須額設定**: 領収書添付を必須とする金額閾値
- **申請期限設定**: 発生日からの申請可能期限（デフォルト2ヶ月）

## 4. 経費カテゴリ定義

| カテゴリコード | カテゴリ名 | 説明 | 月次上限額（目安） | 領収書必須 |
|---|---|---|---|---|
| TRANSPORT | 交通費 | 電車・バス・タクシー等の交通費（通勤費除く） | 設定による | 1万円以上 |
| COMMUTE | 通勤費 | 通勤定期代（月次精算型の場合） | 設定による | 必須 |
| ACCOMMODATION | 宿泊費 | 出張時の宿泊費 | 設定による | 必須 |
| ENTERTAINMENT | 接待交際費 | 顧客・取引先との接待費用 | 設定による | 必須 |
| MEAL | 食事・会議費 | 社内会議・打ち合わせ時の飲食費 | 設定による | 必須 |
| COMMUNICATION | 通信費 | 業務用携帯・インターネット費用 | 設定による | 必須 |
| SUPPLIES | 消耗品・備品 | 業務用文具・消耗品 | 設定による | 1千円以上 |
| BOOKS | 書籍・資料費 | 業務関連書籍・技術書 | 設定による | 必須 |
| TRAINING | 研修・セミナー費 | 業務スキルアップ研修・資格取得費用 | 設定による | 必須 |
| SOFTWARE | ソフトウェア費 | 業務用ソフトウェア・サブスクリプション | 設定による | 必須 |
| OTHER | その他 | 上記に該当しない業務経費 | 設定による | 必須 |

- カテゴリは管理者が追加・編集可能
- 月次上限額は会社ポリシーに基づき管理者が設定
- 軽減税率（8%）対象カテゴリは各カテゴリにフラグ設定

## 7. 他モジュールとの連携インターフェース

### 7.1 Team A（認証・権限）
- SSO/JWT認証によるアクセス制御
- ロール別に参照・操作権限を制御
- 協力会社・個人事業主は自身の申請のみ参照・操作可能

### 7.2 Team B（社員・協力会社）
- 申請者の基本情報（氏名・所属・雇用形態）を参照
- 精算方法（給与計上/別途振込）の設定を参照
- 振込先口座情報を参照（別途振込の場合）

### 7.3 Team C（案件管理）
- 案件IDへの費用配賦（案件別経費管理）
- 案件の予算情報と経費実績の突合（参照のみ）

### 7.4 Team E（通知・ダッシュボード）
- 承認依頼通知（申請提出時）
- 承認・差し戻し・却下通知（申請者へ）
- 精算完了通知（精算処理後）
- 月次ダッシュボードへの経費集計データ提供

## 8. ユースケース

### UC-EXP-001: 社員が交通費を申請する
**アクター**: 社員  
**基本フロー**:
1. EXP-001画面で経費申請を新規作成
2. カテゴリ「交通費」を選択
3. 発生日・金額・利用目的・交通機関詳細を入力
4. 領収書写真をアップロード（スマートフォン撮影画像可）
5. 必要に応じて案件を紐付け
6. 「申請」ボタンをクリック
7. ステータスが「申請中」に変更し、事務へ通知

**代替フロー**:
- 6a. 金額が上限を超えている場合、警告を表示して申請可能（要管理者承認）
- 6b. 領収書未添付で必須金額以上の場合、エラーメッセージを表示

### UC-EXP-002: 接待費申請を承認する
**アクター**: 事務  
**基本フロー**:
1. EXP-004画面で承認待ち一覧を確認
2. EXP-005画面で申請内容・領収書を確認
3. 接待先・目的・金額の妥当性を判断
4. 「承認」ボタンをクリック、コメントを入力
5. 申請者へ承認通知が送信
6. ステータスが「承認済」に変更

**代替フロー**:
- 3a. 内容が不明瞭な場合：「差し戻し」を選択し修正依頼内容を入力

### UC-EXP-003: 月次精算処理を実行する
**アクター**: 事務  
**事前条件**: 対象月の承認済み経費が存在する  
**基本フロー**:
1. EXP-006画面で対象月を選択
2. 精算対象の承認済み経費一覧を確認
3. 社員別・協力会社別の精算額を確認
4. 精算方法（給与計上/別途振込）を確認
5. 「精算実行」ボタンをクリック
6. 精算データ（給与システム連携CSV または 振込データ）が出力される
7. 経費ステータスが「精算済」に変更
8. 対象者へ精算完了通知が送信

## 8A. DDD補足（設計の前提）

本モジュールは「経費管理」コンテキストとして、経費申請〜承認〜精算の業務ルールと状態を所有する。承認タスク/通知は横断基盤（通知・ワークフロー）に委譲し、起点となる状態変化をイベントとして連携する。

### 8A.1 ユビキタス言語（用語集）
- **経費申請（Expense Claim）**: 経費の支出を精算するための申請。下書き〜申請〜承認〜精算の状態を持つ。
- **領収書（Receipt）**: 経費申請に紐づく証憑ファイル。電子帳簿保存法の検索要件を満たすためのメタデータを保持する。
- **承認（Approval）**: 承認者が申請内容を審査し承認/差し戻し/却下を決定する行為。
- **精算（Settlement）**: 承認済み経費を月次でまとめ、給与計上または別途振込として処理すること。
- **上限額（Monthly Limit）**: 申請者×カテゴリ×年月に対する月次上限（適用単位は運用で確定、実装前に明文化する）。
- **閾値（Threshold）**: 領収書必須額や二段階承認の判定に用いる金額境界。

### 8A.2 境界づけ（Bounded Context）と責務
- **本コンテキストが所有する真実（SoT）**
  - 経費申請の状態（ステータス、金額、税率、目的、案件紐付け等）
  - 領収書のメタ情報（ファイル参照、検索用メタ、削除理由等）
  - 月次精算結果（どの年月にいくら精算したか）
- **参照のみ（他コンテキストSoT）**
  - 申請者の基本属性・精算方法・振込先（Team B）
  - 案件/プロジェクト情報（Team C）
  - 承認タスク状態・通知送信履歴（Team E 通知・ワークフロー）

### 8A.3 集約（Aggregate）案
- **ExpenseClaim（経費申請）集約**
  - 子エンティティ: Receipt（領収書）
  - 主な状態: 下書き/申請中/差し戻し/承認済/精算済/却下
- **ExpenseSettlement（精算）集約**
  - 月次×対象者単位で精算合計を確定する（精算対象の申請はスナップショットまたは関連IDで追跡）

### 8A.4 不変条件（ドメインルール）
- **自己承認禁止**: 承認者は申請者本人と異なる必要がある
- **物理削除禁止**: 監査証跡のため論理削除のみ（領収書含む）
- **精算済みの扱い**: 精算済み経費の修正は管理者のみ、修正履歴（理由含む）を必須とする
- **領収書必須**: 領収書必須額以上は領収書なしで提出不可（閾値は設定）

### 8A.5 コマンド（操作）とドメインイベント（状態変化）
- **コマンド例**
  - `SaveDraftExpenseClaim`（下書き保存）
  - `SubmitExpenseClaim`（提出）
  - `RequestChangesExpenseClaim`（差し戻し）
  - `ApproveExpenseClaim`（承認）
  - `RejectExpenseClaim`（却下）
  - `ExecuteMonthlyExpenseSettlement`（月次精算実行）
- **イベント例（連携/監査の起点）**
  - `ExpenseClaimSubmitted`
  - `ExpenseClaimApproved`
  - `ExpenseClaimRejected`
  - `ExpenseClaimSettled`
  - `ExpenseReceiptUploaded`

### 8A.6 ステータス遷移（ステートマシン）
| 現在 | 操作（コマンド） | 次 | 備考 |
|---|---|---|---|
| 下書き | SubmitExpenseClaim | 申請中 | 必須項目/領収書条件を満たすこと |
| 申請中 | RequestChangesExpenseClaim | 差し戻し | 差戻理由必須 |
| 差し戻し | SubmitExpenseClaim | 申請中 | 再提出 |
| 申請中 | ApproveExpenseClaim | 承認済 | 自己承認禁止、閾値により二段階承認あり |
| 申請中 | RejectExpenseClaim | 却下 | 却下理由必須 |
| 承認済 | ExecuteMonthlyExpenseSettlement | 精算済 | 月次精算バッチに紐付く |

## 9. 制約・前提条件

### 9.1 電子帳簿保存法対応
| 要件 | 対応 |
|---|---|
| 真実性の確保 | アップロード日時のタイムスタンプをシステムで記録 |
| 可視性の確保 | PC・スマートフォンからの参照、PDF出力対応 |
| 検索要件 | 日付・金額・取引先名での検索機能を実装 |
| 訂正・削除履歴 | 変更・削除操作の全履歴を記録 |
| 保存期間 | 7年間の保存（法人税法・電子帳簿保存法） |

### 9.2 消費税・インボイス対応
- 経費申請時に消費税率（10%/8%）を選択可能
- 軽減税率（8%）対象品目は対象カテゴリにフラグ設定
- 精算集計時に税率別に集計し、仕入税額控除の管理に活用
- 協力会社からの経費申請は、インボイス登録番号の有無に応じて仕入税額控除の適用を判断

### 9.3 会社規定・内部統制
- 不正申請防止のため、一定金額以上は管理者承認を必須とする
- 承認者は申請者本人と異なる必要がある（自己承認禁止）
- 削除操作は論理削除のみ（物理削除禁止、監査証跡のため）
- 精算済み経費の修正は管理者のみ可能（修正履歴必須）

### 9.4 前提条件
- 社員・協力会社マスタはTeam Bが管理
- 案件マスタはTeam Cが管理
- 給与システムとのリアルタイム連携はスコープ外（CSVエクスポートで対応）
- 協力会社・個人事業主は案件に紐付く経費のみ申請可能（社内経費は申請不可）
