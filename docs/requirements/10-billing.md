# 請求・支払管理 - 要件定義

## 1. モジュール概要

### 1.1 目的
勤怠管理モジュールで確定した勤務データをもとに、顧客への請求書発行・入金管理、および協力会社・個人事業主への支払計算・支払管理を一元的に行う。2023年10月施行のインボイス制度（適格請求書等保存方式）に対応し、消費税処理・免税事業者への対応を含む。

### 1.2 対象ユーザー
| ロール | 主な操作 |
|---|---|
| 事務 | 請求書作成・送付、入金消込、支払処理、月次締め |
| 管理者 | 全データ参照・修正、設定管理、消費税・インボイス設定 |
| 営業 | 担当案件の請求状況参照（読み取り専用） |
| 協力会社/個人事業主 | 請求書提出、支払明細確認 |

### 1.3 スコープ
- 顧客向け請求書の自動生成・発行・管理
- 協力会社・個人事業主への支払計算・支払管理
- 入金消込処理
- 月次締め処理
- インボイス制度対応（適格請求書・免税事業者対応）
- 消費税計算（軽減税率考慮）

---

## 2. 請求フロー（勤務確定→請求書作成→送付→入金確認）

```
勤怠管理モジュール：月次勤務確定データ連携
  ↓
請求計算
  単価 × 実働時間（所定内・時間外・休日出勤の割増計算）
  消費税計算（インボイス登録番号の有無を確認）
  ↓
請求書ドラフト自動生成
  ↓
事務担当者による内容確認・修正
  ↓
管理者承認（金額が閾値以上の場合）
  ↓
顧客への請求書送付
  電子送付（PDF/メール）または 郵送用PDF出力
  ↓
入金待ち（支払期日管理）
  ↓
入金確認・消込処理
  ↓
入金完了（または未入金アラート）
```

---

## 3. 支払フロー（勤務確定→支払計算→支払実行）

```
勤怠管理モジュール：月次勤務確定データ連携
  ↓
支払計算
  契約単価 × 実働時間（所定内・時間外・休日出勤）
  インボイス登録番号の有無を確認
  免税事業者の場合：経過措置に基づく仕入税額控除額計算
  源泉徴収税額計算（個人事業主の場合）
  ↓
支払明細ドラフト自動生成
  ↓
事務担当者による内容確認・承認
  ↓
振込データ生成（全銀フォーマット等）
  ↓
経理システムへのデータ連携（スコープ外：エクスポートのみ）
  ↓
支払実行（銀行振込）
  ↓
支払完了通知（協力会社・個人事業主へ）
```

## 4. 機能要件

### 4.1 顧客向け請求書管理

#### 4.1.1 請求書自動生成
- **トリガー**: 勤怠管理モジュールの月次締め完了後、自動的に請求書ドラフトを生成
- **計算ロジック**:
  - 所定内労働時間 × 基本単価
  - 時間外労働時間 × 時間外単価（契約に基づく割増率）
  - 休日出勤時間 × 休日単価（契約に基づく割増率）
  - 深夜労働時間 × 深夜単価（必要な場合）
  - 合計金額への消費税計算（10%、インボイス対応）
- **対応形式**: SES（人月・時間精算）、請負（固定額・マイルストーン）
- **複数案件**: 1顧客・1月に複数案件がある場合はまとめて請求または案件別請求（設定可）

#### 4.1.2 請求書の記載事項（インボイス制度対応）
適格請求書として以下の記載事項を必須とする：
- 適格請求書発行事業者の氏名または名称
- **登録番号**（T＋13桁の法人番号等）
- 取引年月日
- 取引内容（軽減税率の対象品目である旨）
- 税率ごとに区分した合計対価の額（税抜または税込）
- 適用税率
- **税率ごとに区分した消費税額等**
- 書類の交付を受ける事業者の氏名または名称

#### 4.1.3 請求書管理機能
- **請求書一覧**: 顧客・年月・ステータス（ドラフト/送付済/入金済/未入金/キャンセル）で絞り込み
- **請求書編集**: ドラフト段階で金額・内容の修正が可能
- **承認ワークフロー**: 一定金額以上は管理者承認後に送付（閾値は設定可）
- **PDF出力**: 請求書PDFの出力（インボイス対応フォーマット）
- **電子送付**: PDF添付メールによる送付（送付履歴記録）
- **请求書番号**: 自動採番（形式：INV-YYYY-MM-連番）
- **修正請求書**: 誤りがあった場合の修正請求書発行（元請求書との紐付け必須）
- **前月比較**: 前月請求額との差異アラート（設定値以上の差異）

#### 4.1.4 請求書ステータス管理
| ステータス | 説明 |
|---|---|
| ドラフト | 自動生成・修正中 |
| 承認待ち | 管理者承認待ち |
| 送付済 | 顧客へ送付完了 |
| 入金確認中 | 支払期日到来・確認中 |
| 入金済 | 入金確認・消込完了 |
| 一部入金 | 一部のみ入金 |
| 未入金 | 支払期日超過・未入金 |
| キャンセル | 請求キャンセル（修正請求書発行等） |

### 4.2 協力会社・個人事業主への支払管理

#### 4.2.1 支払計算
- **トリガー**: 勤怠管理モジュールの月次締め完了後、自動計算
- **計算ロジック**:
  - 所定内労働時間 × 契約単価（Team B連携）
  - 時間外労働時間 × 時間外単価（契約の割増率）
  - 休日出勤時間 × 休日単価
  - 合計支払額の計算
  - 消費税計算（登録番号確認・免税事業者対応）
  - 源泉徴収税額計算（個人事業主の場合：支払額の10.21%）

#### 4.2.2 インボイス制度・免税事業者対応
- **適格請求書発行事業者（課税事業者）**: 通常通り消費税を支払い、仕入税額控除が100%適用
- **免税事業者（インボイス未登録）**: 経過措置に基づく控除割合を適用
  - 2023年10月〜2026年9月：仕入税額相当額の80%控除
  - 2026年10月〜2029年9月：仕入税額相当額の50%控除
  - 2029年10月以降：控除不可
- **免税事業者フラグ**: 協力会社・個人事業主マスタにインボイス登録番号の有無を設定（Team B連携）
- **支払明細への明記**: 免税事業者への支払の場合、明細に登録番号なし・経過措置適用の旨を記載

#### 4.2.3 協力会社からの請求書受領（任意）
- **請求書アップロード**: 協力会社・個人事業主がシステム上でPDF等の請求書をアップロード
- **突合確認**: システム計算額とアップロードされた請求書額の突合確認
- **差異アラート**: 一定金額以上の差異がある場合はアラート表示

#### 4.2.4 支払明細管理
- **支払明細一覧**: 協力会社・個人事業主・年月・ステータスで絞り込み
- **支払明細確認**: 協力会社・個人事業主が自身の支払明細を確認可能
- **振込データ出力**: 全銀フォーマット（総合振込）でのデータ出力
- **支払予定日管理**: 支払条件（月末締め翌月末払い等）に基づく支払予定日自動計算

#### 4.2.5 支払ステータス管理
| ステータス | 説明 |
|---|---|
| 計算中 | 自動計算・確認中 |
| 確認待ち | 事務担当者確認待ち |
| 承認済 | 支払承認完了 |
| 振込データ作成済 | 振込ファイル出力済み |
| 支払完了 | 振込完了確認済み |
| 保留 | 差異・確認事項あり |

### 4.3 入金消込

#### 4.3.1 入金登録
- **手動入金登録**: 銀行口座への入金を手動で登録（入金日・金額・振込人名）
- **CSV取込**: 銀行のWeb明細CSVを取り込んで入金データを登録
- **自動消込候補提示**: 入金額と請求額が一致する場合、自動で消込候補を提示

#### 4.3.2 消込処理
- **消込操作**: 入金データと請求書を紐付けて消込
- **部分消込**: 1件の入金が複数請求書に対応する場合の部分消込対応
- **過入金・不足入金**: 差異がある場合は差異金額を表示し、対応方法（返金・翌月調整等）を記録
- **消込取消**: 誤り消込の取消操作（履歴は保持）

#### 4.3.3 未入金管理
- **未入金一覧**: 支払期日超過の請求書一覧
- **督促アラート**: 支払期日の3日前・支払期日当日・超過後にアラート（Team E連携）
- **督促履歴**: 督促連絡の履歴記録

### 4.4 月次締め処理

#### 4.4.1 月次締めの流れ
1. 勤怠管理モジュールから月次確定データを受領済みであることを確認
2. 全請求書の送付状況・入金状況を確認
3. 全支払処理の完了確認
4. 月次集計レポート生成
5. 対象月データをロック（以降は管理者のみ修正可）

#### 4.4.2 月次集計レポート
- 請求合計額・入金合計額・未入金額
- 支払合計額（協力会社・個人事業主別）
- 消費税額合計（課税事業者・免税事業者別）
- 源泉徴収税額合計
- 案件別収支サマリー
- Team Eダッシュボードへのデータ連携

## 7. 他モジュールとの連携インターフェース

### 7.1 Team A（認証・権限）
- SSO/JWT認証によるアクセス制御
- 請求書承認・支払承認は事務・管理者ロールのみ操作可能
- 協力会社・個人事業主は自身の支払明細のみ参照可能

### 7.2 Team B（社員・協力会社）
- 協力会社・個人事業主の契約単価・支払条件を参照
- インボイス登録番号・免税事業者フラグを参照
- 振込先口座情報を参照
- 源泉徴収対象フラグ（個人事業主）を参照

### 7.3 Team C（案件管理）
- 顧客情報・請求条件（請求書送付先・支払条件）を参照
- 案件の契約種別（準委任/請負）・請求単価を参照
- 請求確定情報を案件管理モジュールへ連携（売上確定）

### 7.4 Team E（通知・ダッシュボード）
- 請求書承認依頼・送付完了通知
- 入金期日アラート（期日3日前・期日当日・期日超過）
- 支払完了通知（協力会社・個人事業主へ）
- 月次ダッシュボードへの請求・支払集計データ提供

### 7.5 勤怠管理モジュール（同チームD）
- 月次確定勤務データの受領（トリガー：勤怠締め完了）
  - 受領データ：ユーザーID・案件ID・年月・実働時間詳細（所定内・時間外・休日等）
- 連携後、自動的に請求書・支払明細のドラフトを生成

## 8. ユースケース

### UC-BIL-001: 事務担当者が顧客への請求書を確認・送付する
**アクター**: 事務  
**事前条件**: 勤怠管理モジュールで月次締めが完了し、請求書ドラフトが自動生成されている  
**基本フロー**:
1. BIL-001画面で当月ドラフト請求書一覧を確認
2. BIL-002画面で請求書内容を確認（計算内容・インボイス情報・金額）
3. 必要に応じて金額・明細を修正
4. 「送付」ボタンをクリック（承認が必要な場合は承認依頼）
5. 顧客へPDF添付メールで送付
6. ステータスが「送付済」に変更

**代替フロー**:
- 4a. 金額が承認閾値以上の場合、管理者承認画面へ遷移
- 3a. 内容に誤りがある場合は修正し、修正理由を記録

### UC-BIL-002: 入金消込を行う
**アクター**: 事務  
**事前条件**: 顧客からの入金が銀行口座に確認されている  
**基本フロー**:
1. BIL-014画面で銀行明細CSVを取り込む、またはBIL-004で手動入金登録
2. BIL-004画面でシステムが提示する消込候補を確認
3. 対応する請求書と入金データを選択して消込実行
4. 請求書ステータスが「入金済」に変更

**代替フロー**:
- 3a. 一部入金の場合：消込金額を入力し、差異対応方法を選択・記録
- 3b. 候補が複数ある場合：手動で対応請求書を選択

### UC-BIL-003: 免税事業者の協力会社への支払処理を行う
**アクター**: 事務  
**事前条件**: 月次勤務データが確定し、支払明細ドラフトが生成されている  
**基本フロー**:
1. BIL-006画面で当月支払明細一覧を確認
2. BIL-007画面で免税事業者の支払明細を確認
3. 経過措置に基づく仕入税額控除割合の適用を確認（80%/50%/0%）
4. 内容を確認し「承認」
5. BIL-010画面で振込データ（全銀フォーマット）を出力
6. 銀行システムへ振込データをアップロードして振込実行
7. 支払完了を記録し、協力会社へ支払明細通知

### UC-BIL-004: 月次締め処理を実行する
**アクター**: 事務  
**事前条件**: 当月の全請求書送付および支払処理が完了している  
**基本フロー**:
1. BIL-011画面で対象月を選択
2. 未処理件数（未承認請求書・未振込支払等）を確認
3. 全件完了を確認し「締め処理実行」
4. 月次集計レポートが生成される
5. 対象月データがロック（管理者のみ修正可）
6. Team Eダッシュボードへデータ連携

## 9. 制約・前提条件

### 9.1 インボイス制度（適格請求書等保存方式）対応
| 項目 | 対応内容 |
|---|---|
| 自社の登録番号 | 管理者がシステムに登録し、全請求書に自動印字 |
| 取引先の登録番号確認 | 協力会社・個人事業主マスタに登録番号・免税事業者フラグを管理（Team B） |
| 免税事業者への対応 | 経過措置（80%/50%/0%）を自動計算し支払明細に反映 |
| 記録の保存 | 発行した請求書PDFをシステム内に保存（7年間保存義務） |
| 登録番号の真偽確認 | 国税庁適格請求書発行事業者公表サイトとの突合（手動確認を推奨、自動連携はPhase 2） |

### 9.2 消費税・税率
- 標準税率：10%
- 軽減税率：8%（IT開発サービスは標準税率10%が適用。食料品・定期購読新聞は対象外として設定）
- 消費税の端数処理：切り捨て（請求書単位）

### 9.3 源泉徴収
- 個人事業主（フリーランス）への報酬は源泉徴収が必要
- 源泉徴収税率：10.21%（100万円超の部分は20.42%）
- 支払明細に源泉徴収額を明記し、振込額から控除
- 源泉徴収票の発行はスコープ外（別途経理対応）

### 9.4 請求書・支払記録の保存
- 法人税法上、請求書・支払明細は7年間の保存義務
- システム内にPDFおよびデータとして保存
- 削除操作は論理削除のみ（物理削除禁止）

### 9.5 システム制約
- 月次締め後のデータは管理者のみ修正可能（修正履歴必須）
- 振込データの出力形式：全銀協フォーマット（総合振込）
- 経理システムとのリアルタイム連携はスコープ外（CSVエクスポートで対応）
- 外貨取引はスコープ外（円建てのみ）

### 9.6 前提条件
- 顧客マスタ・契約情報はTeam Cが管理
- 協力会社・個人事業主マスタはTeam Bが管理
- 本モジュールはこれらを参照専用で使用する
- 銀行口座情報はTeam Bが管理し、本モジュールは参照のみ

## 9A. DDD補足（設計の前提）

本モジュールは「請求・支払（Billing & Payment）」コンテキストとして、勤怠確定データを起点に、請求書発行・入金消込・支払計算・月次締め（確定）を行う。帳票（請求書PDF等）および月次締め後のデータは「確定時点の値」を保持し、他コンテキストのマスタ変更で過去の請求・支払結果が変わらないようにする。

### 9A.1 ユビキタス言語（用語集：追加分）
- **請求書ドラフト（Invoice Draft）**: 勤怠確定を起点に自動生成され、送付前に事務が確認・修正できる請求書。
- **送付（Issue/Send）**: 顧客へ請求書を外部に提示する行為。原則として送付後の内容変更は「修正請求書」で行う。
- **入金（Payment Receipt）**: 銀行口座への入金明細。請求書と消込して整合を取る。
- **消込（Matching）**: 入金と請求書の紐付け。部分消込/過不足差異を含む。
- **支払明細（Payment Record）**: 協力会社/個人事業主への支払計算結果。振込データ出力の元となる。
- **月次締め（Close）**: 対象月の請求・入金・支払データをロックし、確定レポートを生成する処理。

### 9A.2 境界づけ（Bounded Context）と責務
- **本コンテキストが所有する真実（SoT）**
  - 請求書（ドラフト〜送付〜入金状態）と、その確定値（番号・金額・税・期日等）
  - 入金データと消込結果（差異対応の記録含む）
  - 支払明細と振込処理状態（振込データ出力/支払完了の記録）
  - 月次締めの結果（ロック状態、月次集計）
- **参照のみ（他コンテキストSoT）**
  - 勤怠の確定勤務データ: Attendance（Team D）
  - 顧客/案件・請求先等: Sales & Project（Team C）
  - 協力会社属性（免税/登録番号/口座等）: People（Team B）
  - 契約条件（単価・割増・支払条件）: Contract（Team D）
  - 承認タスク状態・通知履歴: Workflow & Notification（Team E）

### 9A.3 集約（Aggregate）案
- **Invoice（請求書）集約**
  - 明細（InvoiceItem）を内包し、番号採番・送付・修正請求書発行・入金状態の整合を管理する
- **PaymentRecord（支払明細）集約**
  - 支払計算・免税経過措置・源泉徴収・振込状態の整合を管理する
- **PaymentReceipt（入金）集約**
  - 銀行入金データの登録と、消込（Matching）との整合を管理する
- **MonthlyBillingClose（月次締め）集約**
  - 対象月のロックと確定集計を担い、締め後の変更を制限する

### 9A.4 不変条件（ドメインルール）
- **請求書番号の不変性**: 一度採番した請求書番号は変更不可
- **送付後の変更禁止**: 送付済み請求書の金額・明細の変更は不可。変更が必要な場合は「修正請求書」を発行し、元請求書と紐付ける
- **消込整合性**: 入金の消込合計は入金額を超えない。請求書の入金済金額は消込合計と一致する
- **二重計上防止**: 同一の勤怠確定データから同一請求対象年月・同一条件で重複して請求書/支払明細を確定させない（冪等）
- **締め後ロック**: 月次締め後の対象月データは管理者のみ例外修正可。例外修正は理由・履歴が必須

### 9A.5 コマンド（操作）とドメインイベント（状態変化）
- **コマンド例**
  - `GenerateInvoiceDraftsFromAttendanceClose`（勤怠締めからドラフト生成）
  - `ApproveInvoice` / `SendInvoice`（請求書承認/送付）
  - `IssueCorrectedInvoice`（修正請求書発行）
  - `RegisterPaymentReceipt`（入金登録）
  - `MatchPaymentToInvoices`（消込）
  - `GeneratePaymentRecords`（支払明細生成）
  - `ApprovePaymentRecord` / `ExportBankTransferFile` / `MarkPaymentCompleted`（支払承認/振込データ出力/支払完了記録）
  - `CloseBillingMonth`（月次締め）
- **イベント例（`99-domain-events.md` と整合）**
  - `billing.invoiceIssued`
  - `billing.paymentMatched`
  - `billing.monthClosed`

### 9A.6 ステータス遷移（ステートマシン：請求書）
| 現在 | 操作 | 次 | 備考 |
|---|---|---|---|
| ドラフト | （必要なら）承認依頼 | 承認待ち | 閾値以上のみ |
| 承認待ち | ApproveInvoice | 送付済 | 送付と同時でも可 |
| ドラフト | SendInvoice | 送付済 | 閾値未満は直接送付可（要件で運用確定） |
| 送付済 | RegisterPaymentReceipt + MatchPaymentToInvoices | 入金済 / 一部入金 / 未入金 | 消込結果で更新 |
| 送付済 | IssueCorrectedInvoice | キャンセル（または修正発行済み扱い） | 元請求書と修正請求書を紐付け |
