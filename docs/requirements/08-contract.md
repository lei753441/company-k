# 契約管理 - 要件定義

## 1. モジュール概要

### 1.1 目的
顧客との受注契約（準委任・請負）および協力会社・個人事業主との業務委託契約を一元管理する。契約書の電子化・電子署名連携、更新期限アラート、契約条件の各モジュールへの連携を通じて、契約業務の効率化と法的リスクの低減を実現する。

### 1.2 対象ユーザー
| ロール | 主な操作 |
|---|---|
| 営業 | 顧客契約の起案・管理、案件紐付け |
| 事務 | 契約書管理、電子署名手続き、更新管理 |
| 管理者 | 全データ参照・修正、テンプレート管理 |
| 社員 | 自身が関与する契約の確認（読み取り専用） |
| 協力会社/個人事業主 | 自身との契約内容の確認・電子署名 |

### 1.3 スコープ
- 顧客向け契約（準委任契約・請負契約）の作成・管理
- 協力会社・個人事業主との業務委託契約の作成・管理
- 電子署名サービスとの連携（クラウドサイン等）
- 契約更新期限管理・アラート
- 契約条件（単価・支払条件等）の他モジュールへの連携
- 契約書の電子保存・検索

## 2. 契約種別

### 2.1 顧客向け契約

#### 2.1.1 準委任契約（SES・時間精算型）
- SES（システムエンジニアリングサービス）での人材派遣的な業務に適用
- 月次で実働時間に基づいて精算（時間単価 × 実働時間）
- 所定労働時間・時間外単価・休日単価を契約書に明記
- 瑕疵担保責任なし（成果物ではなく作業を委託）
- 主な契約条件：
  - 業務内容・作業範囲
  - 作業場所（客先/リモート）
  - 精算対象時間（上限・下限）
  - 基本単価（時間単価 or 人月単価）
  - 時間外・休日割増率
  - 支払条件（月末締め翌月末払い等）
  - 契約期間・更新条件

#### 2.1.2 請負契約（受託開発・固定額型）
- システム開発・制作物の完成を目的とした契約
- 成果物の納品をもって請求（マイルストーン払い可）
- 瑕疵担保責任・修正対応義務あり
- 主な契約条件：
  - 業務内容・成果物定義
  - 納期・マイルストーン
  - 請負金額（固定額）
  - マイルストーン別支払スケジュール
  - 支払条件
  - 瑕疵担保期間

### 2.2 協力会社・個人事業主向け契約

#### 2.2.1 業務委託契約（協力会社向け）
- 協力会社への作業委託に適用
- 準委任型（時間精算）または請負型（成果物型）
- 主な契約条件：
  - 業務内容・作業範囲
  - 単価（時間単価 or 人月単価）
  - 時間外・休日割増率
  - 支払条件（月末締め翌月末払い等）
  - 再委託の可否
  - 秘密保持条項
  - 契約期間・更新条件
  - インボイス登録番号（適格請求書発行事業者の場合）

#### 2.2.2 業務委託契約（個人事業主向け）
- フリーランス・個人事業主への作業委託
- 上記に加え以下を明記：
  - 源泉徴収の扱い（源泉徴収義務の確認）
  - インボイス登録番号（未登録の場合は免税事業者として処理）

## 3. 機能要件

### 3.1 契約作成機能

#### 3.1.1 契約書起案
- **テンプレート選択**: 契約種別（準委任/請負/業務委託）に応じたテンプレートを選択
- **基本情報入力**:
  - 契約種別
  - 契約相手（顧客 or 協力会社/個人事業主）
  - 案件への紐付け（Team C連携）
  - 契約期間（開始日・終了日）
  - 単価・支払条件等の契約条件
- **契約書作成**: テンプレートをベースに契約書を作成（テキストエディタまたはWord/PDFアップロード）
- **バージョン管理**: 修正の都度バージョンを記録（v1.0、v1.1等）
- **下書き保存**: 起案途中での下書き保存

#### 3.1.2 テンプレート管理
- **テンプレート登録**: 管理者が契約種別ごとにテンプレートを登録・管理
- **テンプレート変数**: 契約相手名・単価・期間等を変数として定義し、起案時に自動差込
- **バージョン管理**: テンプレート自体のバージョン管理（旧テンプレートで作成した契約書は旧バージョンを参照）

### 3.2 電子署名連携

#### 3.2.1 電子署名フロー
```
事務担当者が署名依頼を送信（クラウドサイン等のAPI連携）
  ↓
契約相手（顧客担当者または協力会社担当者）が電子署名
  ↓
自社担当者が電子署名
  ↓
署名完了・契約書PDF保存
  ↓
契約ステータスを「有効」に変更
```

#### 3.2.2 電子署名サービス連携
- **対象サービス**: クラウドサイン（初期対応）。他サービスはPhase 2以降
- **連携方式**: API連携（署名依頼送信・ステータス確認・完了通知）
- **署名完了後**: 署名済みPDFを自動取得してシステムに保存
- **追跡**: 署名依頼状況（依頼済/署名待ち/完了）をシステム上で確認可能

#### 3.2.3 紙契約書対応
- 電子署名未対応の場合は紙契約書をスキャンしてPDFアップロード対応
- アップロード日時・担当者を記録

### 3.3 更新管理・アラート

#### 3.3.1 更新期限アラート
- **アラートタイミング**: 契約終了日の3ヶ月前・2ヶ月前・1ヶ月前・2週間前
- **通知先**: 担当営業・事務担当者（Team E連携）
- **更新・終了判断**: アラート受信後、更新/終了を入力してアラートを解消

#### 3.3.2 契約更新処理
- **更新バージョン作成**: 既存契約を複製して更新版を作成（単価・期間の変更を反映）
- **変更内容のハイライト**: 前バージョンとの差異を表示
- **更新履歴**: 過去の全バージョンを参照可能

#### 3.3.3 契約終了処理
- **終了記録**: 終了日・終了理由（期間満了/中途解約/その他）を記録
- **関連モジュールへの通知**: 案件管理・勤怠管理・請求管理へ契約終了を通知

### 3.4 契約書管理

#### 3.4.1 契約書一覧
- **一覧表示**: 契約種別・相手方・ステータス・期間で絞り込み検索
- **ステータス管理**: 下書き/署名依頼中/有効/更新待ち/終了/解約のステータス管理
- **期限切れ警告**: 有効期限が近い契約を一覧上でハイライト表示

#### 3.4.2 契約書保存・検索
- **電子保存**: 署名済みPDFをシステム内に保存
- **全文検索**: 契約相手名・案件名・キーワードでの検索
- **ダウンロード**: 契約書PDFのダウンロード（権限に応じて制御）

#### 3.4.3 契約条件の他モジュール連携
- 契約確定後、以下の条件を自動的に連携：
  - 単価・時間外割増率 → 請求・支払管理モジュール
  - 支払条件（締め日・支払サイト） → 請求・支払管理モジュール
  - 所定労働時間・精算時間帯 → 勤怠管理モジュール
  - インボイス登録番号・免税事業者フラグ → Team B（協力会社マスタ）

### 3.5 ステータス定義
| ステータス | 説明 |
|---|---|
| 下書き | 起案中・署名依頼前 |
| 署名依頼中 | 電子署名待ち（いずれかの当事者の署名待ち） |
| 有効 | 全当事者署名完了・契約有効 |
| 更新待ち | 有効期限が近い・更新検討中 |
| 終了 | 契約期間満了・正常終了 |
| 解約 | 契約期間中の中途解約 |
| 無効 | 取消・破棄された契約 |

## 6. 他モジュールとの連携インターフェース

### 6.1 Team A（認証・権限）
- SSO/JWT認証によるアクセス制御
- 契約書の閲覧範囲をロール・担当案件で制御
- 顧客・協力会社は自身に関連する契約のみ閲覧可能

### 6.2 Team B（社員・協力会社）
- 協力会社・個人事業主の基本情報（名称・住所・担当者）を参照
- 契約確定後、以下の条件をTeam Bマスタへ連携：
  - インボイス登録番号・免税事業者フラグ
  - 源泉徴収必要フラグ
  - 単価・支払条件

### 6.3 Team C（案件管理）
- 受注案件を起点として契約書を作成（案件IDで紐付け）
- 顧客情報（請求先・担当者）を案件管理から参照
- 契約ステータス変更（有効化・終了）を案件管理へ通知

### 6.4 Team E（通知・ダッシュボード）
- 更新期限アラート通知（3ヶ月前・2ヶ月前・1ヶ月前・2週間前）
- 署名完了通知（担当営業・事務へ）
- 契約終了・解約通知
- ダッシュボードへの契約数・更新予定件数データ提供

### 6.5 勤怠管理モジュール（同チームD）
- 契約の所定労働時間・精算上下限時間を勤怠管理が参照
- 契約更新・終了時に勤怠管理へ通知（入力可能期間の制御）

### 6.6 請求・支払管理モジュール（同チームD）
- 契約の単価・時間外割増率・支払条件を請求・支払管理が参照
- 契約終了・解約時に請求・支払管理へ通知（該当月以降の請求停止）

### 6.7 電子署名サービス（外部連携）
- **連携サービス**: クラウドサイン（初期対応）
- **連携内容**:
  - 署名依頼の送信（API）
  - 署名ステータスの確認（ポーリングまたはWebhook）
  - 署名完了時のPDF取得（API）
- **認証**: APIキー認証
- **エラーハンドリング**: API接続エラー時は手動対応（紙契約書アップロード）に切り替え可能

## 7. ユースケース

### UC-CON-001: 新規顧客との準委任契約を締結する
**アクター**: 営業、事務  
**事前条件**: Team Cで案件が受注確定している  
**基本フロー**:
1. 営業がCON-003画面で新規契約を起案
2. 契約種別「準委任契約」・案件を選択
3. テンプレートを選択し、契約条件（単価・期間・支払条件等）を入力
4. 契約書ドラフトを保存
5. 事務担当者がCON-004画面で電子署名依頼を送信
6. 顧客担当者がクラウドサイン上で電子署名
7. 自社担当者が電子署名
8. 署名完了・PDFが自動取得されシステムに保存
9. 契約ステータスが「有効」に変更
10. 勤怠管理・請求管理モジュールへ契約条件が連携

**代替フロー**:
- 6a. 顧客が電子署名未対応の場合：紙契約書を郵送し、スキャンしてアップロード

### UC-CON-002: 協力会社（免税事業者）との業務委託契約を締結する
**アクター**: 事務  
**基本フロー**:
1. 事務担当者がCON-003画面で業務委託契約を起案
2. 契約種別「業務委託（協力会社）」・協力会社を選択
3. インボイス登録番号欄を空白のまま、「免税事業者」フラグをONに設定
4. 単価・支払条件等を入力し契約書を作成
5. 電子署名依頼を送信→署名完了
6. 支払管理モジュールが免税事業者として経過措置を適用することを確認

### UC-CON-003: 既存契約の更新処理を行う
**アクター**: 営業、事務  
**事前条件**: 更新期限アラートを受信している  
**基本フロー**:

## 8. 制約・前提条件

### 8.1 法的要件
| 要件 | 内容 |
|---|---|
| 準委任契約 | 民法第656条に基づく委任契約。成果物の完成義務なし。SES業務での適用 |
| 請負契約 | 民法第632条に基づく。成果物の完成義務・瑕疵担保責任（民法637条）あり |
| 業務委託契約 | 偽装請負にならないよう、指揮命令関係の記載に注意（労働者派遣法） |
| 電子署名 | 電子署名法に基づく電子署名による押印代替。本人確認の担保が必要 |
| 収入印紙 | 電子契約書（電磁的記録）は印紙税の課税文書に該当しない（印紙税法基本通達59条） |
| 契約書保存 | 法人税法上、契約書は7年間保存義務 |
| フリーランス保護新法 | 2024年11月施行のフリーランス・事業者間取引適正化等法への対応（発注条件の書面明示等） |

### 8.2 フリーランス保護新法（特定受託事業者への対応）
- 特定業務委託事業者（従業員のいる法人・個人）がフリーランスへ発注する場合：
  - 業務内容・報酬額・支払期日・成果物等を書面（または電磁的方法）で明示する義務
  - 本システムの業務委託契約書（個人事業主向け）にこれらの項目を必須記載として設定
  - 報酬の支払期日：発注した月の翌月末日以内

### 8.3 システム制約
- 契約書データの削除は論理削除のみ（物理削除禁止）
- 署名済み契約書の内容変更は不可（変更が必要な場合は修正契約書または覚書を別途作成）
- 電子署名サービスへのAPI接続が不可な場合は手動（紙・スキャン）でのフォールバック対応
- 契約書PDFは暗号化して保存（ストレージのアクセス制御をTeam Aと連携）

### 8.4 前提条件
- 顧客マスタ・案件マスタはTeam Cが管理
- 協力会社・個人事業主マスタはTeam Bが管理
- 本モジュールは上記マスタを参照のみで管理する
- 電子署名サービス（クラウドサイン）の契約・アカウント管理は別途会社が行う
- 電子署名サービスの利用料はシステム外での精算（スコープ外）
