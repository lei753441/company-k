# 顧客・取引先管理 - 要件定義

## 1. モジュール概要

### 1.1 目的
IT開発会社（SES・受託開発）の営業活動を支援するため、顧客企業・取引先の基本情報、担当者情報、商談履歴を一元管理し、営業の効率化と情報共有を実現する。

### 1.2 対象ユーザー
| ロール | 操作権限 |
|--------|----------|
| 管理者 | 全機能（参照・作成・編集・削除） |
| 営業 | 参照・作成・編集（削除は不可） |
| 事務 | 参照のみ（請求・契約関連の確認目的） |
| 社員 | 参照不可（案件経由で間接的に閲覧） |
| 協力会社/個人事業主 | 参照不可 |

### 1.3 対象データ範囲
- 顧客企業（SES案件・受託開発の発注元）
- 取引先企業（協力会社・パートナー）
- 各企業の担当者情報
- 企業との商談・コミュニケーション履歴

---

## 2. 機能要件

### 2.1 企業管理

#### 2.1.1 企業登録・編集
- 企業情報（社名、住所、電話、URL等）の新規登録・編集
- 企業種別の設定（顧客 / 取引先 / 両方）
- 取引ステータスの設定（見込み / 取引中 / 休眠 / 取引停止）
- 重複企業チェック（法人番号または社名+住所の組み合わせ）
- 企業ロゴ画像のアップロード（任意）

#### 2.1.2 企業検索・絞り込み
- 社名（部分一致・前方一致）での検索
- 企業種別・取引ステータスでの絞り込み
- 業種・地域での絞り込み
- タグによる絞り込み（任意ラベル付け）

#### 2.1.3 企業削除・アーカイブ
- 論理削除（アーカイブ）のみ許可し、物理削除は管理者のみ実施可能
- 紐づく案件・担当者・商談履歴が存在する場合は削除不可（アーカイブのみ可）

### 2.2 担当者管理

#### 2.2.1 担当者登録・編集
- 企業に紐づく担当者情報の登録・編集
- 複数企業への所属設定（兼任対応）
- 役職・部署情報の管理
- 連絡先（メール・電話・その他SNS）の複数登録

#### 2.2.2 担当者検索
- 氏名・メールアドレスでの検索
- 所属企業・役職での絞り込み

#### 2.2.3 担当者削除
- 論理削除のみ。商談履歴が紐づく場合は担当者情報を「退職済み」ステータスへ変更

### 2.3 商談履歴管理

#### 2.3.1 商談記録の登録・編集
- 商談日時・方法（対面 / オンライン / 電話 / メール）の記録
- 自社参加者（営業担当者）と顧客側担当者の記録
- 商談内容・次のアクション・メモのフリーテキスト入力
- ファイル添付（議事録・提案資料等）
- 商談に関連する案件のひも付け（任意）

#### 2.3.2 商談履歴の検索・閲覧
- 企業・担当者・期間・担当営業での絞り込み
- タイムライン形式での表示

#### 2.3.3 リマインダー・フォローアップ
- 次回アクション日時の設定とリマインダー通知（メール / システム内通知）
- 未フォローアップ商談のアラート表示

## 6. ユースケース

### UC-CUS-01: 新規顧客企業の登録
**アクター:** 営業  
**事前条件:** 営業ロールでログイン済み  
**主フロー:**
1. 企業一覧画面から「新規登録」を選択
2. 企業登録フォームに必須項目（社名、企業種別、ステータス）を入力
3. 重複チェックが実行され、既存企業との重複なしを確認
4. 保存を実行し、企業詳細画面へ遷移
5. 担当者を追加登録する

**代替フロー:**
- 3a. 重複企業が検出された場合、候補リストを表示し既存企業との統合または新規登録を選択させる

---

### UC-CUS-02: 商談後の履歴記録
**アクター:** 営業  
**事前条件:** 対象企業が登録済み  
**主フロー:**
1. 企業詳細画面から「商談記録を追加」を選択
2. 商談フォームに日時・方法・内容・次のアクションを入力
3. 関連案件をひも付け（任意）
4. 必要に応じてファイル（議事録等）を添付
5. 保存し、タイムラインに表示されることを確認

---

### UC-CUS-03: フォローアップ管理
**アクター:** 営業  
**事前条件:** フォローアップ日が設定済みの商談履歴が存在する  
**主フロー:**
1. フォローアップ管理画面を開く
2. 本日以前の未対応アクション一覧を確認
3. 対応済みのものをクローズ、未対応のものを更新

## 7. 制約・前提条件

### 7.1 データ制約
- 企業の物理削除は管理者権限のみ許可（監査証跡の保持のため）
- 商談履歴は削除不可（修正のみ可）
- 法人番号が入力された場合、国税庁法人番号APIとの連携による自動補完（将来対応）

### 7.2 権限制約
- 営業担当者は自分が登録した企業のみ編集可能（管理者は全編集可能）とするか、チーム共有にするかはビジネスルールに従い決定すること
- 顧客情報は社員・協力会社には非公開とする

### 7.3 性能要件
- 企業一覧の初期表示：2秒以内（1,000件以上を想定）
- 検索結果の表示：1秒以内

### 7.4 前提条件
- Team A（認証・権限）のユーザー管理が完成していることを前提とする
- 企業登録時の重複チェックのルールは運用開始前に確定すること
