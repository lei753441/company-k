# 勤怠管理 - 要件定義

## 1. モジュール概要

### 1.1 目的
社員および協力会社・個人事業主の勤務実績を正確に記録・集計し、労働基準法・36協定に準拠した勤怠管理を実現する。確定した勤務データを請求・支払管理モジュールへ連携することで、月次の請求・支払処理を自動化する。

### 1.2 対象ユーザー
| ロール | 主な操作 |
|---|---|
| 社員 | 勤怠打刻、月次勤務表入力・提出、残業・休暇申請 |
| 協力会社/個人事業主 | 月次勤務報告書の入力・提出 |
| 事務 | 勤務表承認、集計、修正、月次締め |
| 管理者 | システム設定、マスタ管理、全データ参照・修正 |
| 営業 | 担当案件の勤務状況参照（読み取り専用） |

### 1.3 スコープ
- 社員の日次打刻管理（出勤・退勤・休憩）
- 月次勤務表の入力・提出・承認ワークフロー
- 協力会社・個人事業主の月次勤務報告管理
- 残業・休暇申請ワークフロー
- SES案件における客先勤怠システムとのデータ連携（CSV取込）
- 36協定・労働基準法に基づく上限管理・アラート
- 確定勤務データの請求・支払管理モジュールへの連携

## 2. 勤務表フロー（打刻/入力→提出→承認→確定）

### 2.1 社員（自社オフィス勤務）
```
日次打刻（出勤・退勤・休憩）
  ↓
月次勤務表自動生成（打刻データから）
  ↓
社員による確認・修正・コメント入力
  ↓
社員が勤務表を提出（月末締め日までに）
  ↓
事務/管理者が承認
  ↓
勤務確定 → 請求・支払管理モジュールへデータ連携
```

### 2.2 社員（SES・客先常駐）
```
客先勤怠システムで打刻（客先管理）
  ↓
月次勤務表を本システムへ入力 または CSV取込
  ↓
社員による確認・コメント入力
  ↓
社員が勤務表を提出（月末締め日までに）
  ↓
事務/管理者が承認
  ↓
勤務確定 → 請求・支払管理モジュールへデータ連携
```

### 2.3 協力会社/個人事業主
```
月次勤務報告書を本システムへ入力 または CSV取込
  ↓
協力会社担当者が提出（月末締め日までに）
  ↓
事務/管理者が承認
  ↓
勤務確定 → 支払管理モジュールへデータ連携
```

### 2.4 ステータス定義
| ステータス | 説明 |
|---|---|
| 入力中 | 勤務データ入力中（提出前） |
| 提出済 | 社員/協力会社が提出完了 |
| 差し戻し | 事務/管理者が修正指示 |
| 承認済 | 事務/管理者が承認 |
| 確定 | 月次締め処理完了・請求支払連携済み |

## 3. 機能要件

### 3.1 社員向け機能

#### 3.1.1 日次打刻
- **出勤打刻**: システムログイン後に出勤時刻を記録。ブラウザベース（PCおよびスマートフォン対応）
- **退勤打刻**: 退勤時刻を記録
- **休憩打刻**: 休憩開始・終了を記録（複数回対応）
- **打刻修正申請**: 打刻漏れ・誤りがあった場合、理由を添えて修正申請。事務承認後に反映
- **打刻確認**: 当日および過去の打刻記録を確認

#### 3.1.2 月次勤務表
- **勤務表表示**: 月次カレンダー形式で日ごとの勤務時間・種別を表示
- **手動入力**: SES客先常駐社員は客先での実績を手動入力
- **CSV取込**: 客先勤怠システムから出力されたCSVを取り込み（後述のフォーマット仕様参照）
- **勤務表提出**: 入力完了後に提出操作。提出後は編集不可（差し戻し時は再編集可）
- **提出期限アラート**: 締め日の3日前・前日にシステム通知（Team E連携）

#### 3.1.3 残業申請
- **事前申請**: 残業予定日・予定終了時刻・理由を入力し申請
- **事後申請**: やむを得ず事前申請なく残業した場合、翌営業日までに申請
- **申請ステータス確認**: 申請中・承認済・差し戻しの確認

#### 3.1.4 休暇申請
- **休暇種別選択**: 有給休暇・特別休暇・慶弔休暇・欠勤等を選択
- **日程入力**: 開始日・終了日（半休の場合は午前/午後）
- **残日数確認**: 有給残日数をリアルタイム表示
- **申請・取消**: 申請および承認前の取消操作

### 3.2 協力会社/個人事業主向け機能

#### 3.2.1 月次勤務報告書
- **日次入力**: 日ごとの作業開始時刻・終了時刻・休憩時間・作業内容を入力
- **CSV取込**: 指定フォーマットのCSVアップロードによる一括入力
- **勤務報告書提出**: 入力完了後に提出。提出後は編集不可（差し戻し時は再編集可）
- **提出期限**: 案件ごとに設定された締め日（デフォルト：月末）
- **提出履歴確認**: 過去の勤務報告書の確認

#### 3.2.2 勤務報告書の項目
- 対象年月
- 案件（プロジェクト）
- 日付・曜日
- 作業開始時刻・終了時刻
- 休憩時間（分）
- 実働時間（自動計算）
- 作業内容/備考
- 合計実働時間（自動集計）
- 所定内時間・時間外時間（契約上の所定労働時間との差分から自動計算）

### 3.3 事務・管理者向け機能

#### 3.3.1 勤務表・報告書承認
- **一覧表示**: 提出済み勤務表・報告書の一覧（絞り込み：月・担当者・ステータス・案件）
- **承認操作**: 内容確認後に承認。コメント入力可
- **差し戻し操作**: 修正理由を入力して差し戻し。差し戻し時は社員/協力会社へ通知（Team E連携）
- **一括承認**: 複数件を選択して一括承認

#### 3.3.2 勤務データ集計
- **月次集計表示**: 社員・協力会社別の月次勤務時間集計
- **残業時間集計**: 月次・年次の残業時間集計。36協定上限との比較表示
- **プロジェクト別集計**: 案件・プロジェクトごとの工数集計（Team C連携）
- **CSV出力**: 集計データのCSVエクスポート

#### 3.3.3 勤務データ修正
- **管理者修正**: 承認後のデータを管理者権限で修正。修正履歴（誰が・いつ・何を修正したか）を記録
- **修正理由必須**: 修正時は理由入力を必須とする

#### 3.3.4 月次締め処理
- **締め処理実行**: 対象月の全データが承認済みであることを確認し、締め処理を実行
- **締め処理後**: データが確定状態となり、請求・支払管理モジュールへ自動連携

#### 3.3.5 マスタ管理（管理者）
- **祝日マスタ**: 年間祝日カレンダーの登録・更新
- **締め日設定**: 案件・協力会社ごとの勤務報告締め日設定
- **所定労働時間設定**: 社員・雇用形態別の所定労働時間設定
- **有給付与管理**: 社員ごとの有給日数付与・消化履歴管理

## 4. 勤務種別定義

| コード | 種別名 | 説明 | 備考 |
|---|---|---|---|
| NORMAL | 通常勤務 | 所定労働時間内の勤務 | |
| OVERTIME | 時間外残業 | 所定労働時間を超えた時間外労働（平日） | 1.25倍割増 |
| MIDNIGHT | 深夜残業 | 22:00〜5:00の労働 | 1.25倍割増（通常残業との重複時1.5倍） |
| HOLIDAY_WORK | 休日出勤 | 法定休日（日曜等）の労働 | 1.35倍割増 |
| LEGAL_HOLIDAY_WORK | 法定休日出勤 | 週1日の法定休日の労働 | 1.35倍割増 |
| PAID_LEAVE | 有給休暇 | 年次有給休暇 | 労基法39条 |
| HALF_PAID_LEAVE_AM | 午前半休 | 午前中の有給休暇 | 0.5日消化 |
| HALF_PAID_LEAVE_PM | 午後半休 | 午後の有給休暇 | 0.5日消化 |
| SPECIAL_LEAVE | 特別休暇 | 慶弔・育児・介護等の特別休暇 | 種別別に設定 |
| ABSENCE | 欠勤 | 無断・病欠等の欠勤 | 給与控除対象 |
| COMPENSATORY | 代休 | 休日出勤に対する代替休日 | |
| REMOTE | リモートワーク | 在宅勤務（勤務時間は通常と同じく記録） | フラグで識別 |

## 7. 他モジュールとの連携インターフェース

### 7.1 Team A（認証・権限）
- SSO/JWT認証によるログイン
- ロールベースアクセス制御（RBAC）でロール別に画面・機能を制御
- 打刻操作時のIPアドレス・デバイス情報記録（不正打刻防止）

### 7.2 Team B（社員・協力会社）
- 社員・協力会社の基本情報（氏名・所属・雇用形態）参照
- 所定労働時間・契約形態（SES/受託）参照
- 有給付与情報の管理連携

### 7.3 Team C（案件管理）
- 案件IDを勤務記録に紐付け（プロジェクト別工数管理）
- 案件ごとの締め日・所定労働時間設定を参照
- 確定工数をTeam Cへ通知（WBS進捗管理に利用）

### 7.4 Team E（通知・ダッシュボード）
- 勤務表提出期限アラート（締め日3日前・前日）
- 承認・差し戻し通知（メール/システム内通知）
- 残業上限超過アラート通知
- 月次ダッシュボード向けデータ提供（勤務時間統計等）

### 7.5 請求・支払管理モジュール（同チームD）
- 月次締め完了後、確定勤務データを連携
  - 連携データ：ユーザーID・案件ID・年月・総実働時間・時間外時間・休日出勤時間
- 連携タイミング：月次締め処理完了時（APIまたはDB経由）

### 7.6 客先勤怠システム（外部連携）
- CSV取込フォーマット（標準）：
  - 日付、出勤時刻、退勤時刻、休憩時間（分）、実働時間、備考
  - 文字コード：UTF-8またはShift-JIS（設定可能）
  - 日付フォーマット：YYYY/MM/DD または YYYY-MM-DD
- 取込時にバリデーションエラーがあった場合は一覧表示し、修正を促す
- 複数フォーマットへの対応は案件ごとのマッピング設定で対応

## 8. ユースケース

### UC-ATT-001: 社員が月次勤務表を提出する
**アクター**: 社員  
**事前条件**: 該当月の日次打刻または手動入力が完了している  
**基本フロー**:
1. 社員がATT-002画面を開く
2. 月次勤務表の内容を確認・コメントを入力
3. 「提出」ボタンをクリック
4. システムが未入力日・異常値（深夜退勤翌日早朝出勤等）を警告
5. 社員が確認して提出を確定
6. ステータスが「提出済」に変更
7. 事務担当者へ承認依頼通知（Team E連携）

**代替フロー**:
- 3a. 締め日を過ぎている場合は警告を表示（管理者権限で強制提出可能）

### UC-ATT-002: 事務担当者が勤務表を承認する
**アクター**: 事務  
**事前条件**: 社員から勤務表が提出されている  
**基本フロー**:
1. 事務担当者がATT-009画面で提出済み一覧を確認
2. ATT-010画面で対象者の勤務表を詳細確認
3. 残業時間・36協定との整合性を確認
4. 「承認」ボタンをクリックし、コメントを入力
5. ステータスが「承認済」に変更
6. 社員へ承認通知（Team E連携）

**代替フロー**:
- 4a. 問題がある場合は「差し戻し」を選択し理由を入力
- 4b. 社員へ差し戻し通知が送信され、ステータスが「差し戻し」に変更

### UC-ATT-003: 協力会社が勤務報告書をCSVで取込・提出する
**アクター**: 協力会社/個人事業主  
**基本フロー**:
1. ATT-016画面でCSVファイルをアップロード
2. システムがフォーマットを検証してプレビュー表示
3. 内容を確認し、取込を実行
4. ATT-007画面に取込データが反映される
5. 追加コメントを入力し「提出」
6. 事務担当者へ承認依頼通知

### UC-ATT-004: 月次締め処理を実行する
**アクター**: 事務  
**事前条件**: 該当月の全勤務表・報告書が承認済み  
**基本フロー**:
1. ATT-012画面で対象月を選択
2. 未承認件数・未提出件数を確認
3. 全件承認済みを確認し「締め処理実行」
4. システムが集計データを確定し、ステータスを「確定」に変更
5. 請求・支払管理モジュールへデータ連携
6. 締め処理完了通知を関係者へ送信

## 9. 制約・前提条件

### 9.1 法的要件
| 要件 | 詳細 |
|---|---|
| 労働基準法 第32条 | 法定労働時間：1日8時間・1週40時間を超える場合は36協定が必要 |
| 36協定 月次上限 | 時間外労働の上限：原則月45時間（特別条項あり最大100時間未満）をシステムで管理・アラート |
| 36協定 年次上限 | 年間360時間（特別条項あり最大720時間）をシステムで管理 |
| 深夜割増 | 22:00〜5:00の労働に対して25%割増賃金計算対象であることを記録 |
| 休日割増 | 法定休日労働に対して35%割増の記録 |
| 有給年5日取得義務 | 年10日以上付与者への年5日取得義務（労基法39条7項）管理・アラート |
| 勤怠記録の保存 | 賃金台帳等として5年間の保存義務（労基法109条） |
| 勤怠記録の客観性 | タイムカード・ICカード等客観的方法による記録（厚労省指針）。手入力修正時は修正履歴必須 |

### 9.2 システム制約
- 打刻はPCブラウザおよびスマートフォンブラウザ対応（ネイティブアプリは対象外）
- 打刻時刻はサーバー時刻を使用（クライアント時刻は使用しない）
- 月次勤務表の確定（ロック）後は管理者以外編集不可
- 削除操作は論理削除のみ（物理削除禁止、監査証跡のため）

### 9.3 SES固有の前提条件
- 客先に常駐するSES社員は、客先の勤怠システムで打刻が完了しているケースが多い
- 本システムへはCSV取込または手動入力で登録する
- 客先CSVフォーマットは案件ごとに異なる場合があり、マッピング設定で対応する
- 客先勤怠システムとのリアルタイム連携はスコープ外（Phase 2以降の検討）

### 9.4 前提条件
- 社員マスタ・協力会社マスタはTeam Bが管理し、本モジュールは参照のみ
- 案件情報はTeam Cが管理し、本モジュールは案件IDを参照して紐付ける
- 所定労働時間は雇用形態・案件ごとに異なる場合があり、管理者が設定する
