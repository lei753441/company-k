# スキル・キャリア管理 - 要件定義

**バージョン**: 1.0  
**作成日**: 2026-04-26  
**担当チーム**: Team B

---

## 1. モジュール概要

### 1.1 目的

SES・受託開発会社の社員が保有するスキル・資格・職務経歴を一元管理し、以下の用途に活用する。

1. **営業活動支援**: スキルシートの迅速な作成・提出（客先常駐案件の応募）
2. **案件マッチング**: スキル情報をマッチングエンジン（Team C）に提供し、適切な案件への配置を支援
3. **人材育成**: スキルギャップの可視化とキャリアパス管理
4. **稼働状況管理**: 現在の稼働状況・空き状況の把握

### 1.2 スコープ

| 対象 | 内容 |
|------|------|
| 対象ユーザー | 管理者、営業、事務、社員本人 |
| 管理対象 | 自社社員のスキル・資格・経歴（正社員・契約社員） |
| 対象外 | 協力会社・個人事業主のスキル管理（モジュール05で管理） |

### 1.3 主要機能サマリ

- スキルシートの作成・編集・バージョン管理
- 技術スキルの登録（言語・FW・DB・クラウド・ツール等）
- 資格・認定の管理（取得日・有効期限・証明書管理）
- 職務経歴（プロジェクト経歴）の管理
- 稼働状況・空き予定の管理
- スキルシートPDF出力（Team E連携）
- マッチングエンジンへのスキルデータ提供（Team C連携）

## 2. 機能要件

### 2.1 スキルシート管理

**概要**: SES業界標準のスキルシート（職務経歴書）を作成・管理する。客先への提出用PDFとして出力できる。

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| SKL-001 | スキルシート作成 | 社員ごとのスキルシートを新規作成する |
| SKL-002 | スキルシート編集 | スキルシートの内容を更新する |
| SKL-003 | スキルシートバージョン管理 | 編集のたびにバージョン履歴を保持し、過去バージョンへの参照を可能にする |
| SKL-004 | スキルシートPDF出力 | 客先提出用フォーマットでPDF出力する（Team E連携） |
| SKL-005 | スキルシート共有リンク | 有効期限付き閲覧専用リンクを発行する（外部共有用） |
| SKL-006 | スキルシートテンプレート管理 | 複数のPDF出力テンプレートを管理する（案件種別ごとのフォーマット対応） |

**スキルシート構成要素**:
1. 基本プロフィール（氏名・最終学歴・業務経験年数）
2. 技術スキル一覧（言語・FW・DB・クラウド等と経験年数・習熟度）
3. 資格・認定一覧
4. プロジェクト経歴一覧（新しい順）
5. 自己PR・その他特記事項

### 2.2 技術スキル管理

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| SKL-010 | スキル登録 | 社員が保有するスキルをカテゴリ・タグ付きで登録する |
| SKL-011 | スキル編集 | 経験年数・習熟度レベルを更新する |
| SKL-012 | スキル削除 | スキルエントリを削除する |
| SKL-013 | スキルマスタ管理 | 管理者がスキルマスタ（タグ・カテゴリ）を追加・編集する |
| SKL-014 | スキルタグサジェスト | 入力中にスキルマスタからサジェストを表示する |

**習熟度レベル定義**:

| レベル | 名称 | 定義 |
|--------|------|------|
| 1 | 入門 | 学習中・業務経験なし |
| 2 | 初級 | 指導のもと業務で使用可能（経験1年未満） |
| 3 | 中級 | 独力で業務で使用可能（経験1〜3年） |
| 4 | 上級 | 他者を指導できるレベル（経験3年以上） |
| 5 | エキスパート | 技術的な意思決定・アーキテクチャ設計ができるレベル |

### 2.3 資格・認定管理

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| SKL-020 | 資格登録 | 取得済み資格を登録する |
| SKL-021 | 資格更新 | 更新情報・有効期限を更新する |
| SKL-022 | 資格有効期限アラート | 有効期限90日前・30日前に本人・管理者にメール通知する |
| SKL-023 | 証明書ファイル管理 | 資格証明書PDFをアップロード・管理する |
| SKL-024 | 資格マスタ管理 | 資格マスタを管理者が追加・編集する |

### 2.4 職務経歴（プロジェクト経歴）管理

**概要**: SES・受託案件での業務経験を案件単位で記録・管理する。

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| SKL-030 | 案件経歴登録 | 参加した案件（社内・客先）の経歴を登録する |
| SKL-031 | 案件経歴編集 | 経歴情報を編集する |
| SKL-032 | 案件連携 | Team Cの案件モジュールと連携し、アサイン終了案件を経歴に自動追加する（確認フロー付き） |
| SKL-033 | 担当役割管理 | 各案件でのロール（PM・PL・SE・PG等）を記録する |
| SKL-034 | 経歴並び替え | スキルシート上での案件表示順を手動並び替えする |

### 2.5 稼働状況管理

**概要**: SES特有の稼働状況（客先常駐中・社内待機・空き予定）を管理し、営業活動・マッチングに活用する。

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| SKL-040 | 稼働ステータス更新 | 現在の稼働状況を更新する |
| SKL-041 | 空き予定登録 | アサイン終了予定日・空き開始予定日を登録する |
| SKL-042 | 稼働状況一覧 | 全社員の稼働状況を一覧表示する（営業・管理者向け） |
| SKL-043 | 空き社員アラート | 空き状態または空き予定30日以内の社員を営業に通知する |

**稼働ステータス定義**:

| ステータス | 説明 |
|-----------|------|
| assigned_client | 客先常駐中（SESアサイン中） |
| assigned_internal | 社内プロジェクト担当中 |
| available | 待機中（即日アサイン可） |
| available_soon | 空き予定あり（終了日が確定） |
| on_leave | 休職中 |
| training | 研修・自己学習中 |

### 3.3 スキルマスタ管理ポリシー

- スキルタグは管理者のみ追加・廃止できる
- 廃止されたタグは既存の社員スキルには残るが、新規入力ではサジェストされない
- スキルタグは定期的（四半期ごと）に見直しを行う
- 社員からのスキルタグ追加申請は管理者が承認後に反映する

## 6. マッチングモジュールへのインターフェース（Team C連携）

### 6.1 スキル検索API

Team Cのマッチングエンジンがスキル情報を検索するためのAPIを提供する。

**エンドポイント**: `GET /api/v1/skills/search`

**リクエストパラメータ**:

| パラメータ | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| skill_ids | Array[BIGINT] | 任意 | スキルマスタIDの配列 |
| min_proficiency | INT | 任意 | 習熟度レベルの下限（1〜5） |
| min_experience_years | DECIMAL | 任意 | 経験年数の下限 |
| availability_statuses | Array[ENUM] | 任意 | 稼働ステータスフィルタ |
| available_from | DATE | 任意 | 空き開始日の上限 |

**レスポンス例**:
```json
{
  "employees": [
    {
      "employee_id": "EMP-20240401-0001",
      "name": "山田 太郎",
      "skills": [
        {
          "skill_id": 101,
          "skill_name": "Java",
          "proficiency_level": 4,
          "experience_years": 5.0
        }
      ],
      "availability": {
        "status": "available_soon",
        "available_from_date": "2026-06-01"
      }
    }
  ],
  "total": 1
}
```

### 6.2 社員スキル詳細取得API

**エンドポイント**: `GET /api/v1/skills/{employee_id}`

マッチング候補として選定された社員の詳細スキル情報を返す。

**レスポンス**: スキル一覧・資格一覧・経歴サマリ・稼働状況を含む

### 6.3 稼働状況通知（イベント）

稼働ステータスが変更された際に、Team Cにイベントを送信する。

| イベント名 | タイミング | 内容 |
|-----------|-----------|------|
| availability.updated | 稼働ステータス変更時 | employee_id・新ステータス・空き開始予定日 |
| assignment.ended | アサイン終了時 | Team Cからの通知受信後、稼働ステータスを自動更新 |

### 6.4 Team Eへのスキルシートデータ提供

| インターフェース | 内容 |
|-----------------|------|
| `GET /api/v1/skillsheets/{employee_id}/data` | PDF生成に必要なスキルシートの全データを返す |
| `GET /api/v1/skillsheets/{employee_id}/pdf` | PDF生成をTeam Eに委譲し、生成URLを返す |

## 7. ユースケース

### UC-SKL-01: スキルシートの作成・更新

**アクター**: 社員本人、事務担当  
**前提条件**: 社員が本システムにアカウントを持っていること

**主フロー**:
1. 社員が自身のスキルシート画面（SCR-SKL-002）にアクセスする
2. スキル編集画面（SCR-SKL-003）に遷移する
3. スキルタグをカテゴリから選択または検索し、習熟度・経験年数を入力する
4. プロジェクト経歴を追加する（案件名・期間・役割・業務内容・使用技術）
5. 「保存」ボタンをクリックする
6. 新バージョンが自動採番され保存される
7. スキルシートプレビュー（SCR-SKL-004）で確認する

## 8. 制約・前提条件

### 8.1 性能要件

| 項目 | 要件 |
|------|------|
| スキル検索API応答時間 | 1秒以内（マッチングエンジンからの呼び出し） |
| スキルシート一覧表示 | 3秒以内（最大500件） |
| PDF生成（Team E経由） | 30秒以内 |

### 8.2 データ整合性

- 社員が退職した場合、スキルシートデータは保持するが編集不可にする
- スキルマスタのタグを廃止した場合、既存の社員スキルデータのタグは残す（表示時に「廃止済みスキル」と明示）
- プロジェクト経歴は、Team Cから連携された案件データを変更した場合、連携元には影響しない（コピーとして管理）

### 8.3 セキュリティ要件

- スキルシートの共有リンクは有効期限（最大30日）を設定し、発行者のみが無効化できる
- 客先名は非公開設定が可能で、PDF出力時には「某社」等に変換する
- 社員本人は自身のスキルシートのみ編集可能。他社員のスキルシートは閲覧のみ

### 8.4 前提条件

- 社員マスタ（モジュール02）が先行して構築されていること
- スキルマスタの初期データが投入されていること
- Team C（案件マッチングモジュール）のAPIスキーマが確定していること
- PDF生成テンプレートはTeam Eと共同で設計すること

### 8.5 運用上の制約

- スキルシートの編集は本人・事務・管理者に限定し、営業は閲覧のみ
- 資格証明書ファイルのストレージ容量上限: 社員1人あたり500MB
- スキルシートバージョン履歴は最大50バージョンを保持（超過分は古いバージョンから削除）
