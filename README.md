## 概要（スモールスタート構成）
本システムはAWSのマネージドサービス中心に構成し、**小さく始めて段階的に拡張**できることを優先します。

## アーキテクチャ
- **フロントエンド**: React（静的ホスティング）
  - **S3**: 静的ファイル配置
  - **CloudFront**: CDN配信（OACでS3直アクセスは遮断）
- **バックエンド**: API Gateway + AWS Lambda
  - **API Gateway**: HTTP API（認可連携、ルーティング）
  - **Lambda**: ビジネスロジック実行
- **認証**: Amazon Cognito
  - サインイン/トークン発行（JWT）
  - APIはJWTを前提に認可（権限設計はアプリ側で拡張可能）
- **データベース**: Amazon DynamoDB
  - アクセスパターン先行でテーブル/GSIを設計
- **ファイル保存**: Amazon S3
  - 画像・添付などのオブジェクトを保存
  - アップロードは**事前署名URLでクライアント→S3直接PUT**を基本（API経由アップロードは避ける）

## ねらい
- **運用を軽く**（サーバ管理なし、従量課金）
- **スケール容易**（トラフィック増に追随しやすい）
- **セキュアなデフォルト**（CloudFront経由、JWT認証、最小権限）

## 最小の運用・観測（推奨）
- **CloudWatch Logs**: Lambda / API Gatewayログ
- （必要に応じて）**WAF**: CloudFrontまたはAPI Gateway前段
